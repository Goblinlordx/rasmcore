# Repository Hygiene

Rules for keeping the git repository clean, small, and fast to clone.

## No Binary Files

- **Never commit binary files larger than 10 KB** to git
- Test fixture images (.raw, .png, .jpg, .tiff, .webp, .gif, .bmp, .dds, .fits) must be generated programmatically
- Use existing generator patterns: `gradient_rgb()`, `solid_rgb()`, `make_animated_gif()`, `make_multipage_tiff()`
- If a fixture absolutely must come from an external tool (e.g., dcraw reference output), keep it under 10 KB or document how to regenerate it
- **Exception**: `font_8x16.bin` (1.5 KB embedded bitmap font) — too small to matter

## No Generated Code

- **`bindings.rs`** is auto-generated by `cargo-component` / `wit-bindgen` — **NEVER commit**
- **`generated_*.rs`** files in `$OUT_DIR` — **NEVER commit** (they're in the build output directory)
- **`param-manifest.json`** is build-time output — **NEVER commit**
- If a file has "Auto-generated" or "Do not edit" in its header, it should not be in git
- The `.gitignore` has patterns for all known generated files — respect them

## Build Artifacts

- `*.wasm` files belong in `target/` or a gitignored output directory
- `demo/sdk/` builds are ephemeral — built with `demo/build.sh`, gitignored
- `node_modules/` — never committed
- Cargo build outputs (`target/`) — never committed

## Test Fixtures

### Adding New Test Data

1. **Prefer programmatic generation**: Write a function that creates the test data from parameters
2. **Use `LazyLock`** for expensive fixtures that multiple tests share:
   ```rust
   static GRADIENT_128: LazyLock<Vec<u8>> = LazyLock::new(|| {
       let mut data = Vec::with_capacity(128 * 128 * 3);
       for y in 0..128u32 {
           for x in 0..128u32 {
               data.push((x * 255 / 127) as u8);
               data.push((y * 255 / 127) as u8);
               data.push(128);
           }
       }
       data
   });
   ```
3. **For external reference outputs** (ImageMagick, OpenCV, dcraw):
   - If under 10 KB: commit with a comment explaining how to regenerate
   - If over 10 KB: generate on test run, cache in a gitignored directory
   - Document the external tool command in the test or a README

### Existing Generator Patterns

| Pattern | Location | Use For |
|---------|----------|---------|
| `gradient_rgb(w, h)` | reference_audit.rs | Test gradients |
| `make_gradient_rgb(w, h)` | pro_filter_parity.rs | Parity test inputs |
| `solid_rgb(w, h, r, g, b)` | filter tests | Solid color images |
| `make_animated_gif()` | decoder tests | Multi-frame GIF |
| `make_multipage_tiff()` | decoder tests | Multi-page TIFF |
| `perlin_noise(w, h, seed, ...)` | filter generators | Procedural noise |

## .gitignore Patterns

The `.gitignore` must include at minimum:
```
/target/
**/bindings.rs
*.wasm
demo/sdk/
node_modules/
tests/fixtures/generated/
tests/fixtures/.venv/
*.raw
```

## Pre-commit Check

Run `scripts/check-repo-hygiene.sh` before committing to catch violations:
- Detects staged binary files > 10 KB
- Detects staged `bindings.rs` files
- Detects staged `*.wasm` files
- Detects staged `*.raw` files

Install as a git hook (optional):
```bash
cp scripts/check-repo-hygiene.sh .git/hooks/pre-commit
chmod +x .git/hooks/pre-commit
```
